# AI-Powered Developer Copilot Platform Constitution

## Project Vision

Build a repository-aware, production-grade SaaS platform that acts as an AI engineering assistant for software teams.

The platform combines capabilities of GitHub Copilot, CodeRabbit, SonarQube, Jira Assistant, and ChatGPT into a single system.

Its purpose is to help developers:

* Understand codebases
* Review code
* Detect bugs
* Detect security vulnerabilities
* Generate documentation
* Generate tests
* Manage technical debt
* Refactor code
* Interact with repositories through AI-powered chat

## Non-Negotiable Rules

1. Preserve the project vision.
2. Preserve the multi-agent architecture.
3. Preserve the repository-aware RAG architecture.
4. Never convert the platform into a CRUD application.
5. Never remove scalability considerations.
6. Never remove security requirements.
7. Never remove observability requirements.
8. Never remove audit logging requirements.
9. Never remove multi-tenancy requirements.
10. Never remove RBAC requirements.
11. Every implementation decision must move the project closer to a production-grade SaaS platform.
12. All architectural changes must be justified and documented.
